2. The Trading Strategy Explained

The core of this EA is the London Session Liquidity Sweep with a Change in State of Delivery (CISD) entry model. It's designed to capitalize on institutional order flow during the high-volume London Killzone.

Key Concepts

  • Asian Range: The high and low established during the relatively quiet Asian trading session. This range acts as a pool of liquidity.
  • Liquidity Sweep: Price action that pushes above the Asian high or below the Asian low, triggering stop-loss orders and attracting breakout traders. This is often an institutional maneuver to engineer liquidity.
  • Change in State of Delivery (CISD): The critical confirmation signal. After a liquidity sweep, a CISD occurs when price aggressively reverses and breaks through a recent Fair Value Gap (FVG), signaling a potential shift in market direction. This broken FVG becomes an "Inversion FVG".
  • Entry Points: After a CISD, the EA looks for a high-probability entry by waiting for a retracement into one of three zones: the Inversion FVG, a newly formed FVG, or a classic Order Block.

Bullish Scenario

  1. Price sweeps below the Asian session low.
  2. A bullish candle closes above a prior bearish FVG, confirming a bullish CISD.
  3. The EA waits for price to retrace and enters a BUY trade at an Inversion FVG, a new bullish FVG, or a bullish Order Block.
  4. Stop Loss: Placed below the low of the entry structure (FVG or Order Block).
  5. Take Profit: Targets the Previous Day's High (PDH) or a fixed Risk:Reward ratio.

Bearish Scenario

  1. Price sweeps above the Asian session high.
  2. A bearish candle closes below a prior bullish FVG, confirming a bearish CISD.
  3. The EA waits for price to retrace and enters a SELL trade at an Inversion FVG, a new bearish FVG, or a bearish Order Block.
  4. Stop Loss: Placed above the high of the entry structure (FVG or Order Block).
  5. Take Profit: Targets the Previous Day's Low (PDL) or a fixed Risk:Reward ratio.

Using the web interface is a simple three-step process: Configure, Generate, and Copy.

  2. Generate MQL5 Code: Click the "Generate MQL5 Expert Advisor" button. The AI will write the complete code based on your inputs.
  3. Copy Code: Once the code appears in the right-hand panel, click the "Copy Code" button.

4. Deploying the EA in MetaTrader 5

Follow these steps to get your generated EA running in MT5.

  1. Open MetaEditor: In MT5, go to Tools > MetaQuotes Language Editor or press F4.
  2. Create New File: In MetaEditor, go to File > New. In the MQL5 Wizard, select Expert Advisor (template) and click Next.
  3. Name Your EA: Give it a name (e.g., LondonICT_Strategy) and click Finish.
  4. Paste the Code: A template file will open. Select all the text (Ctrl+A) and delete it. Then, paste your copied code (Ctrl+V).
  5. Compile the Code: Click the Compile button in the toolbar or press F7. You should see "0 error(s), 0 warning(s)" in the toolbox at the bottom.
  6. Attach to Chart: Go back to the MT5 terminal. In the "Navigator" window, find your EA under "Expert Advisors". You may need to right-click and "Refresh". Drag the EA onto the chart you wish to trade.
  7. Configure Inputs: In the pop-up window, review the "Inputs" tab. All the settings you chose in the generator are here, but you can tweak them. In the "Common" tab, ensure "Allow Algo Trading" is checked. Click OK.
  8. Activate Algo Trading: In the main MT5 toolbar, make sure the master "Algo Trading" button is enabled (green). The EA name will appear in the top-right of the chart with a green icon if it's running correctly.

5. Expert Advisor Inputs (In MT5)

When you attach the EA to a chart, you can configure the following parameters in the "Inputs" tab.

Parameter Description
LotSize The fixed lot size for each trade. Set to 0 to use RiskPercentage instead.
RiskPercentage The percentage of your account balance to risk per trade. Only active if LotSize is 0.
AsianSession_Start_Hour The server hour when the Asian session begins.
AsianSession_End_Hour The server hour when the Asian session ends.
LondonSession_Start_Hour The server hour when the London Killzone begins (EA starts looking for trades).
LondonSession_End_Hour The server hour when the London Killzone ends (EA stops looking for new trades).
TakeProfit_Mode Method for setting the take profit: targeting PDH/PDL or a fixed Risk:Reward ratio.
EnableDailyStopLoss If true, the EA will stop trading for the day after reaching MaxDailyLosses.
MaxDailyLosses The number of losing trades allowed per day before the EA pauses.
TrailToBreakEven If true, moves the stop loss to the entry price once a certain profit is reached.
BreakEvenRR The Risk:Reward ratio (e.g., 1.0 for 1:1) that must be hit to trigger the move to break-even.
EnableTrailingStop If true, activates a dynamic trailing stop to lock in profits.
TrailingStopPips The distance in pips the stop loss will trail behind the current price.
ShowSessionTimers If true, displays a countdown timer for trading sessions on the chart.

The EA draws several objects on the chart to help you visualize its logic:

  • Asian Range: A rectangle showing the high and low of the session.
  • Entry Zones: Colored rectangles will appear when the EA identifies a valid FVG or Order Block for a potential trade.
  • Pending SL/TP Lines: Dotted horizontal lines will show the calculated Stop Loss and Take Profit levels for a valid setup before entry.
  • Session Timers: A label in the corner of the chart shows countdowns to the next session start or end time.
